## Key Method
Two changes: (1) Keep tied embedding in FP16 instead of int8 quantization -- the embedding pulls double duty as output head, so every bit of precision matters. This drops post-quant BPB degradation from ~0.007 to ~0.0005. (2) Tuned LR schedule: WARMDOWN_ITERS 1200->3600, MATRIX_LR 0.04->0.06. The default schedule assumes more steps than you get in 10 minutes.

Tradeoff: FP16 embed costs ~500KB extra, so MLP hidden reduced from 1024 to 992.

## Components Changed
- Architecture: yes -- MLP hidden 1024->992 to offset FP16 embed size
- Quantization: yes -- FP16 passthrough for tok_emb.weight instead of int8
- Training: yes -- WARMDOWN_ITERS=3600, MATRIX_LR=0.06
- Evaluation: no -- standard non-overlapping eval
- Compression: no -- still zlib
- Token features: no
- Initialization: no

## Reported Metric
val_bpb: 1.2197 (seed 1337), 1.2201 (seed 42)
Pre-quant vs post-quant gap: ~0.0005 BPB (baseline gap was ~0.007)
Improvement over baseline: ~0.005 bpb

## Likely Mechanism
The tied embedding is by far the most quantization-sensitive tensor because errors affect both input representation AND output logit accuracy. FP16 preserves this critical pathway. The LR/warmdown tuning better matches the actual training duration.
